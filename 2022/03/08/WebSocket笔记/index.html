3. 建立连接

  1. 直接@ServerEndpoint("/imserver/{userId}") 、@Component启用即可，然后在里面实现@OnOpen开启连接，@onClose关闭连接，@onMessage接收消息等方法

  2. 新建一个ConcurrentHashMap webSocketMap 用于接收当前userId的WebSocket，方便IM之间对userId进行推送消息。单机版实现到这里就可以。

  3. 集群版（多个ws节点）还需要借助mysql或者redis等进行处理，改造对应的sendMessage方法即可。

  4. @ServerEndpoint这个是WebSocket注解，这个类级别的注释声明它装饰的类是一个 Web 套接字端点，它将被部署并在 Web 套接字服务器的 URI 空间中可用。注释允许开发人员定义该端点将发布的 URL（或 URI 模板），以及端点的其他重要属性到 websocket 运行时，例如它用于发送消息的编码器。

    带注释的类必须有一个公共的无参数构造函数。

    value = “/message/getMsgCount”是前端发布websocket的URL地址，如果无法访问走免认证路径

  5. WebSocket启动的时候优先于spring容器，从而导致在WebSocketServer中调用业务Service会报空指针异常

    所以需要在WebSocketServer中将所需要用到的service给静态初始化一下：

    private static IThreatMessageService messageService;

4. 消息推送

小铃铛方式推送消息个数，前端展开小铃铛查看消息列表（注意userId是连接的用户id，调用sendMessage会推送给传入的userId用户）
